## 第一章 执行摘要

### 1.1 一句话描述

**OpenBook 是一个 AI Agent 驱动的品味发现与消费决策平台——每个用户拥有一个懂自己品味的 AI 分身，它在真实世界中为你发现、评测、推荐"对味"的好东西，并帮你做出更好的消费决策。**

### 1.2 核心价值主张

| 传统模式 | OpenBook 模式 |
|---------|-------------|
| 你主动搜索、看评论、比价 | AI Agent 主动发现并推荐 |
| 看大众平均评分 | 基于**你的品味偏好**个性化评价 |
| 被营销内容和虚假种草左右 | Agent 无利益冲突，基于品味客观评价 |
| 信息过载，决策疲劳 | Agent 过滤噪音，精准匹配 |
| 消费和社交分离 | 品味相投的人自然聚集 |

### 1.3 商业模式核心

```
用户的 AI Agent → 发现好东西 → 生成个性化评测 → 用户感兴趣
                                                      ↓
                                               点击购买/到店
                                                      ↓
                                            电商平台完成交易
                                                      ↓
                                          OpenBook 获得 CPS 分佣
```

本质上，OpenBook 是一个由 AI 驱动的新一代"消费决策引擎"，变现模式是 **CPS 导购分佣 + 会员订阅 + 品牌合作** 的三重收入结构。

### 1.4 关键数据

| 指标 | 数值 |
|------|------|
| 目标市场规模（本地生活 + 电商导购） | 2.5 万亿+（2025 年中国本地生活服务市场） |
| 对标公司营收（什么值得买，2024） | 15.2 亿元/年 |
| 目标用户画像 | 18-35 岁城市年轻消费者 |
| 潜在用户池 | 3.5 亿+（小红书月活规模的消费决策人群） |
| MVP 状态 | ✅ 已完成，已部署上线 |

## 第二章 市场分析

### 2.1 宏观市场

**消费决策链条正在发生根本性变化：**

```
传统路径：需求产生 → 搜索引擎/电商搜索 → 看评价 → 比价 → 下单
社交路径：社交媒体种草 → 搜索验证 → 下单（当前主流）
AI 路径：AI Agent 主动推荐 → 用户确认 → 一键下单（正在发生）
```

**关键市场数据：**

- **2025 年中国本地生活服务市场规模超 2.5 万亿元**（艾媒咨询）
- **线上渗透率仅 30.8%**（艾瑞咨询），仍有巨大增长空间
- **小红书月活 3.5 亿**，1.7 亿用户每月有购买意向搜索
- **小红书日均搜索量近 6 亿次**（2024 Q4）
- **2024 年全球 AI 领域融资总额达 5,995 亿元**，同比翻倍
- Perplexity 于 2024 年 11 月推出 AI Shopping，估值 90 亿美元
- 什么值得买上线 AI 购物助手"小值"，单月调用量环比增长 50%
- OpenAI 于 2025 年推出 ChatGPT Shopping 功能

### 2.2 消费趋势

麦肯锡和英敏特报告揭示：

1. **理性消费崛起**：从"冲动种草"转向"精明决策"
2. **情价比消费**：Z 世代追求情绪价值与价格的最大比值（同比增长 16.2%）
3. **品质升级**：44.8% 的年轻人关注品质升级，并非"消费降级"
4. **信任危机**：虚假种草导致用户对传统推荐体系的信任度下降
5. **AI 接受度高**：Z 世代对 AI 辅助消费决策接受程度最高

**核心洞察：消费者需要一个"无利益冲突、懂我的消费顾问"——这正是 AI Agent 最适合的角色。**

### 2.3 市场机会定位

市场存在一个明确的空白地带——**高度个性化 × 高度可信赖 × AI 原生**：

- **小红书**：个性化但充满营销内容，信任度在下降
- **什么值得买**：可信但不够个性化，基于大众推荐
- **大众点评**：可信但只覆盖本地生活，评分同质化
- **Perplexity Shopping**：AI 驱动但未进入中国市场

### 2.4 市场规模测算

| 层级 | 定义 | 规模 |
|------|------|------|
| **TAM** | 中国线上消费决策相关 GMV | ~2.5 万亿/年 |
| **SAM** | 通过导购/种草平台影响的消费决策 | ~3,000 亿/年 |
| **SOM** | 早期聚焦品质消费品（咖啡、3C、美食等） | ~200 亿/年 |
| **年收入目标（3年）** | SOM × CPS 佣金率 3%-5% | 6-10 亿/年（天花板） |

## 第三章 产品定位与愿景

### 3.1 使命

> 让每个人都拥有一个懂自己品味的 AI 消费顾问，在信息过载的时代，帮你发现真正"对味"的好东西。

### 3.2 愿景（3-5 年）

> 成为中国年轻消费者首选的 AI 消费决策平台——当你想知道"买什么"、"去哪吃"、"什么值得体验"时，第一个打开 OpenBook。

### 3.3 核心理念：品味即身份

1. **品味是个人的** —— 没有"客观最好"，只有"最适合你的"
2. **AI 比营销更诚实** —— Agent 无利益驱动，只基于品味理解
3. **好品味会传染** —— 品味相投的人互相发现，形成正向社区
4. **消费决策值得更好** —— 从信息噪音中解放，享受发现的乐趣

### 3.4 品牌调性

| 维度 | 描述 |
|------|------|
| 视觉风格 | 简洁、温暖、有质感（Notion × 小红书） |
| 语言风格 | 年轻但不浮夸，有观点但不说教 |
| 情感定位 | "发现"的惊喜感 + "被理解"的共鸣感 |
| Slogan 候选 | "你的品味，AI 懂" / "让 AI 替你发现好东西" / "品味相投，自然相遇" |

## 第四章 用户画像与场景

### 4.1 三类核心用户

**用户 A：探索型小白领**（占比 50%）
- 26 岁，互联网运营，上海，月入 15-25K
- 每周尝试新店/新品，在小红书种草但担心被割韭菜
- 痛点："不知道信谁" / "搜索比价太累"
- 期望："帮我从海量信息中找到真正适合我的"

**用户 B：品味达人**（占比 25%）
- 30 岁，设计师，深圳，月入 25-40K
- 对咖啡/音响/相机有深度研究，关注小众品牌
- 痛点："我的品味很小众，大众排行对我没意义"
- 期望："找到真正懂我品味的人和社群"

**用户 C：效率决策者**（占比 25%）
- 33 岁，产品经理，北京，月入 30-50K
- 工作忙没时间研究，但追求品质，愿为节省时间付费
- 痛点："没时间刷小红书，但又不想随便买"
- 期望："告诉我什么值得买就行"

### 4.2 四大核心场景

**场景 1：品味发现（日常 Feed 浏览）**
Agent 主动推送："发现一家评分很高的手冲咖啡店，离你 800 米"

**场景 2：消费决策（主动搜索/对话）**
用户问："2000 元左右的降噪耳机？" → Agent 返回 3 个基于品味偏好的精选推荐 + 价格对比 + 一键购买链接

**场景 3：品味社交**
发现品味相似度 89% 的用户 → 查看品味画像 → 关注 → 品味圈交流

**场景 4：品味日报（被动推送）**
每日 Agent 生成：新开店铺、关注商品降价、品味圈热议话题

## 第七章 商业模式与收入结构

### 7.1 四大收入来源

#### 收入源 1：CPS 导购佣金（占比 50-60%）

用户因 Agent 推荐而购买 → 平台获得佣金。这是最自然最健康的模式。

| 渠道 | 品类 | 佣金率 | 客单价 | 单笔佣金 |
|------|------|--------|--------|---------|
| 淘宝联盟 | 日用/美妆/食品 | 5-15% | ¥80 | ¥4-12 |
| 淘宝联盟 | 数码/家电 | 1-5% | ¥500 | ¥5-25 |
| 京东联盟 | 3C/家电 | 1-5% | ¥800 | ¥8-40 |
| 饿了么 | 外卖 | 6%+ | ¥35 | ¥2+ |
| 多多进宝 | 日用百货 | 10-40% | ¥30 | ¥3-12 |

关键 API：`taobao.tbk.item.get`（商品详情）、`taobao.tbk.tpwd.create`（淘口令生成）、`taobao.tbk.order.details.get`（订单佣金查询）

#### 收入源 2：Pro 会员订阅（占比 20-30%）

| 功能 | 免费 | Pro |
|------|------|-----|
| Agent 日推荐 | 5 条 | 无限 |
| AI 对话搜索 | 3 次/天 | 无限 |
| 品味日报 | 简版 | 完整 + 深度分析 |
| 月度品味报告 | ❌ | ✅ 可分享 |
| 价格追踪 | 3 件 | 50 件 |
| 降价提醒 | ❌ | ✅ |
| Agent 深度定制 | 基础 | 完整品味训练 |
| 去广告 | ❌ | ✅ |

#### 收入源 3：品牌合作（占比 15-20%）

| 合作类型 | 定价 |
|---------|------|
| 品味评测（Agent 推荐商品露出） | CPM ¥30-80 |
| 品牌专题（深度评测 3-5 篇） | ¥5-20 万/期 |
| 品味人群包（脱敏消费者洞察） | ¥2-10 万/份 |
| 年度合作（入驻 + 数据 + 专属页面） | ¥30-100 万/年 |

**核心卖点：** OpenBook 的品味标签可以帮品牌精准触达——不是"25 岁女性"，而是"偏好手冲咖啡、关注设计感、预算 200-500 的精品消费者"。

#### 收入源 4：数据洞察服务（远期，占比 5-10%）

品味趋势报告、选品建议、AI 市场调研（替代传统市调）。

### 7.2 收入模式对标

| 平台 | 主要收入 | 2024 年营收 |
|------|---------|------------|
| 什么值得买 | CPS + 广告 + 代运营 | 15.2 亿元 |
| 小红书 | 广告 + 电商 | ~260 亿（估算） |
| Perplexity | 订阅 + 商户 | ~1 亿美元（估算） |

## 第九章 竞品分析与差异化

### 9.1 竞品定位矩阵

- **左下（传统 × 大众化）：** 大众点评、淘宝搜索
- **左上（AI × 大众化）：** ChatGPT Shopping、Perplexity Shopping
- **右下（传统 × 个性化）：** 小红书、什么值得买、得物
- **右上（AI × 个性化）：** ★ OpenBook 的目标位置

### 9.2 关键差异化

**vs 小红书：** AI 原生 + 无广告干扰 + 品味深度个性化
**vs 什么值得买：** 品味导向（非价格导向）+ Agent 原生
**vs 大众点评：** 个性化评分 + 跨品类 + 电商整合
**vs Perplexity Shopping：** 中国本土化 + 品味深度 + 社交属性

### 9.3 竞争壁垒

1. **品味数据资产** —— 越用越精准，迁移成本高
2. **Agent 个性化** —— 每用户不同，不可复制
3. **信任链网络效应** —— 品味社交粘性
4. **CPS 场景闭环** —— 内容+数据+购买，变现效率高

## 第十章 获客与增长策略

### 10.1 阶段策略

**种子期（0 → 1K DAU）：** Product Hunt/即刻发布、品味社群精准推广、KOC 邀请体验、小红书病毒式测试内容（"AI 测你什么品味风格"）

**增长期（1K → 10K DAU）：** 可分享品味报告（类 Spotify Wrapped 病毒传播）、品味测试 H5、Agent 评测内容 SEO、邀请得 Pro 天数、小红书/抖音内容投放

**规模期（10K → 100K DAU）：** ASO 优化、付费投放（¥50K/月）、品牌联合推广、科技/消费媒体 PR

### 10.2 增长飞轮

```
品味内容质量提升 → 用户留存提升 → 用户购买转化 → CPS 收入增长
     ↑                                                  ↓
 Agent 数据更多 ← 新用户加入 ←──────── 增长投入加大
```

### 10.3 关键留存策略

| 策略 | 目标 |
|------|------|
| 品味日报推送 | D7 留存 > 30% |
| 降价提醒 | 唤醒率 > 15% |
| 品味成就/徽章 | 月活参与 > 40% |
| 社交互动通知 | 社交留存加成 > 20% |
| Agent 进化（越用越懂你） | D30 留存 > 20% |

## 第十二章 风险分析与应对

### 12.1 风险矩阵

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 电商平台调低佣金率 | 中 | 高 | 多平台分散；增加会员/品牌收入占比 |
| LLM 成本上升 | 低 | 中 | 使用 Deepseek 等性价比模型；缓存减少调用 |
| 用户对 AI 推荐不信任 | 中 | 高 | 透明展示推荐逻辑；UGC 补充；可解释性 |
| 大厂做类似功能 | 高 | 中 | 深耕品味个性化壁垒；社交网络效应 |
| 数据合规风险 | 低 | 高 | 严格遵守《个保法》；不爬取第三方数据 |
| AI 内容质量问题 | 中 | 中 | 人工审核 + 用户反馈 + 内容安全策略 |
| 增长不及预期 | 中 | 高 | 低成本运营；灵活调整获客策略 |

### 12.2 最大风险：大厂跟进

**应对分析：**

大厂（小红书/美团/淘宝）有可能在自己的平台内添加 AI 推荐功能。但 OpenBook 的壁垒在于：

1. **跨平台中立性**：大厂的 AI 推荐只会推自己平台的商品/商家，OpenBook 可以跨平台比较
2. **品味深度**：大厂的推荐算法优化"点击率"和"GMV"，OpenBook 优化"品味匹配度"
3. **社交层**：品味社交是大厂功能追加难以复制的——需要独立的社区氛围
4. **数据独立性**：用户的完整品味画像散落在各平台，只有 OpenBook 能跨平台聚合
